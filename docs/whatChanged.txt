v4.0.0 — Complete Rust Rewrite (AIAudit)

- Renamed project from AIHound to AIAudit; binary is now `aiaudit`
- Complete rewrite from Python/Go to Rust; zero runtime dependencies
- All 29 scanners ported to Rust with full platform support (Linux/macOS/Windows/WSL)
- Added remote Windows fleet scanning via SMB2 + Service Control Manager (PSExec-style)
  - NTLMv2 auth: hand-rolled MD4/HMAC-MD5 SPNEGO, no external crates
  - Kerberos auth (--features kerberos): GSSAPI via libgssapi 0.11 with no-kinit path
    (gss_acquire_cred_with_password; --user/--password work directly)
  - Uploads aiaudit.exe to ADMIN$\Temp, runs via transient SCM service, cleans up
  - Parallel execution with rayon (--parallel flag)
  - Inventory file support: JSON, YAML, CSV, plain text
  - Active Directory host discovery via LDAP (--ad-server / --ad-base-dn)
- Added interactive web dashboard (docs/report.html): Chart.js, no server required
- Added docs/sample_results.json for dashboard testing
- Removed Python source, Go source, PyInstaller build artifacts
- Removed --bloodhound / BloodHound export (not implemented in v4; see BLOODHOUND_GUIDE.md)
- Removed --mcp / MCP server mode (not implemented in v4)
- Removed --banner, --watch-log, --notify, --notify-min-risk, --min-risk, --debounce flags
- Simplified watch mode: --watch and --interval only
- Feature flags: default (local), remote (adds ldap3), kerberos (adds libgssapi, includes remote)
- Git history squashed to single clean initial commit
- Published to https://github.com/sbergs/AIAudit
